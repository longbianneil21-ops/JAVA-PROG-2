INTERMEDIATE PROGRAMMING (CC103)
WEEK 2: JAVA PROGRAMMING REVIEW
DATA TYPES
Variables are reserved memory locations to store values. When you create a variable, you reserve space in memory. Based on the data type of a variable, the operating system allocates memory and decides what can be stored. By assigning different data types to variables, you can store integers, decimals, or characters.
There are two data types available in Java: primitive data types and reference/object data types.
PRIMITIVE DATA TYPES
Primitive types are the most basic data types available within the Java language. There are eight primitive data types supported by Java. They are predefined by the language and named by a keyword.
The eight primitive data types are: byte, short, int, long, float, double, boolean, an






d char.
* Byte stores whole numbers from negative one hundred twenty-eight to one hundred twenty-seven.
* Short stores whole numbers from negative thirty-two thousand seven hundred sixty-eight to thirty-two thousand seven hundred sixty-seven.
* Int stores whole numbers from negative two billion one hundred forty-seven million four hundred eighty-three thousand six hundred forty-eight to two billion one hundred forty-seven million four hundred eighty-three thousand six hundred forty-seven.
* Long stores whole numbers from negative nine quintillion two hundred twenty-three quadrillion three hundred seventy-two trillion thirty-six billion eight hundred fifty-four million seven hundred seventy-five thousand eight hundred eight to nine quintillion two hundred twenty-three quadrillion three hundred seventy-two trillion thirty-six billion eight hundred fifty-four million seven hundred seventy-five thousand eight hundred seven.
* Float stores fractional numbers, sufficient for six to seven decimal digits.
* Double stores fractional numbers, sufficient for fifteen decimal digits.
* Boolean stores true or false values.
* Char stores a single character or letter, or ASCII values.
REFERENCE DATA TYPES
Reference variables are created using defined constructors of classes. They are used to access objects. These variables are declared to be of a specific type that cannot be changed. Examples include employees, puppies, and so on.
Class objects and various types of array variables come under reference data type. The default value of any reference variable is null. A reference variable can be used to refer to any object of the declared type or any compatible type.
Examples of reference types include array, class, interface, string, applet, message box, input box, and HiLo.
JAVA LITERALS
A literal is a source code representation of a fixed value. Literals are represented directly in the code without any computation. Literals can be assigned to any primitive type variable. We can specify a literal to char data type as a single character within single quotes.
STREAMS
A stream is a sequence of objects that supports various methods which can be pipelined to produce the desired result. Streams support many different kinds of data, including simple bytes, primitive data types, localized characters, and objects. Some streams simply pass on data; others manipulate and transform the data in useful ways.
CONDITIONAL STATEMENTS
Conditional statements in a computer program support decisions based on a certain condition. If the condition is met, or true, a certain piece of code is executed.
If-Then and If-Then-Else Statement
The most basic flow control statement in Java is if-then: if something is true, do something. This statement is a good choice for simple decisions. The basic structure of an if statement starts with the word "if," followed by the statement to test, followed by curly braces that wrap the action to take if the statement is true. This statement can also be extended to do something else if the condition is false.
Nested-If
A nested-if is an if statement that is the target of another if or else. Nested-if statements mean an if statement inside another if statement.
Switch-Case
A multiway branch statement. It provides an easy way to dispatch execution to different parts of code based on the value of the expression.
LOOPS
Looping in programming languages is a feature which facilitates the execution of a set of instructions or functions repeatedly while some condition evaluates to true. Java provides three ways for executing loops: while loop, for loop, and do-while loop.
While Loop
A while loop is a control flow statement that allows code to be executed repeatedly based on a given Boolean condition. The while loop can be thought of as a repeating if statement.
For Loop
The for loop provides a concise way of writing the loop structure. Unlike a while loop, a for statement consumes the initialization, condition, and increment or decrement in one line, thereby providing a shorter, easy-to-debug structure of looping.
Do-While Loop
The do-while loop is similar to the while loop with the only difference that it checks for the condition after executing the statements, and therefore is an example of an exit control loop.
ARRAYS
An array is a group of like-typed variables that are referred to by a common name. Arrays in Java work differently than they do in C or C++.
Important points about Java arrays: In Java, all arrays are dynamically allocated. Since arrays are objects in Java, we can find their length using the object property length. This is different from C or C++ where we find length using sizeof. A Java array variable can be declared like other variables with square brackets after the data type. The variables in the array are ordered, and each has an index beginning from zero. Java arrays can also be used as a static field, a local variable, or a method parameter. The size of an array must be specified by an int value and not long or short. Arrays can contain primitives as well as object references of a class depending on the definition of the array. In the case of primitive data types, the actual values are stored in contiguous memory locations. In the case of objects of a class, the actual objects are stored in heap segments.
To declare an array, declare the variable type with square brackets. To insert values, you can use an array literal by placing the values in a comma-separated list inside curly braces.

ENUM
An enum type is a special data type that enables a variable to be a set of predefined constants. The variable must be equal to one of the values that have been predefined for it. Common examples include compass directions (north, south, east, and west) and the days of the week.
In the Java programming language, you define an enum type by using the enum keyword. The constants are separated by commas and are typically written in uppercase letters.
BIT SET
BitSet is a class defined in the java.util package. It creates an array of bits represented by boolean values. The size of the array is flexible and can grow to accommodate additional bits as needed. Because it is an array, the bit values can be accessed by non-negative integers as an index. The interesting aspect of BitSet is that it is easy to create and manipulate bit sets that basically represent a set of boolean flags.
The BitSet class provides two constructors: a no-argument constructor to create an empty BitSet object, and a one-constructor with an integer argument to represent the number of bits in the BitSet.
The class also provides methods for common bit manipulation using bitwise logical AND, bitwise logical OR, and bitwise logical exclusive OR. For example, you can use and, or, and xor methods. The methods include: and, andNot, cardinality, clear, clone, equals, flip, and others.
VECTOR
Vector is like a dynamic array which can grow or shrink its size. Unlike an array, you can store n-number of elements in it as there is no size limit. It is a part of the Java Collection framework since Java 1.2. It is found in the java.util package and implements the List interface, so you can use all the methods of the List interface.
It is recommended to use the Vector class only in thread-safe implementations. If you do not need thread-safe implementation, you should use ArrayList because it performs better. Vector is synchronized. Java Vector contains many legacy methods that are not part of the collections framework.
Constructors include: an empty vector with default size ten, a vector with specified initial capacity, a vector with specified initial capacity and capacity increment, and a vector that contains the elements of a collection.
STACKS
A Stack is a Last In First Out (LIFO) data structure. It supports two basic operations called push and pop. The push operation adds an element at the top of the stack, and the pop operation removes an element from the top of the stack.
Methods include: testing if the stack is empty, returning the element on the top without removing it, returning the element on the top and removing it, pushing an element onto the stack, and searching for an element in the stack (returning its offset from the top or negative one if not found).
DICTIONARY
A Java dictionary is an abstract class that stores key-value pairs. Given a key, its corresponding value can be stored and retrieved as needed; thus, a dictionary is a list of key-value pairs. The Dictionary object classes are implemented in java.utils. Dictionary is an abstract class, representing a key-value relation and works similar to a map. The sole constructor is the default Dictionary constructor.
HASHTABLE
This class implements a hash table, which maps keys to values. Any non-null object can be used as a key or as a value. To successfully store and retrieve objects from a hashtable, the objects used as keys must implement the hashCode method and the equals method. It is similar to HashMap but is synchronized. Hashtable stores key-value pairs in a hash table. A key is hashed, and the resulting hash code is used as the index at which the value is stored within the table.
Constructors include: a default constructor, a constructor that creates a hash table with an initial size, a constructor that creates a hash table with an initial size and a fill ratio (which determines how full the hash table can be before it is resized, with a value between zero point zero and one point zero), and a constructor that creates a hash table initialized with the elements of a map.
PROPERTIES
Properties is a subclass of Hashtable. It is used to maintain lists of values in which the key is a String and the value is also a String. The Properties class is used by many other Java classes. For example, it is the type of object returned by System.getProperties() when obtaining environmental values.
Constructors include: a constructor that creates a Properties object with no default values, and a constructor that creates an object that uses a default properties list for its default values. In both cases, the property list is empty.

WEEK 5: COLLECTIONS
COLLECTION
Java collections refer to a collection of individual objects that are represented as a single unit. A collection is an object that can hold references to other objects. The collection interfaces declare the operations that can be performed on each type of collection. The classes and interfaces of the collections framework are in the package java.util.
JAVA COLLECTION FRAMEWORK
The Java Collection Framework offers the capability to represent a group of elements in classes and interfaces. A Java collection framework includes interfaces, classes, and algorithms.
The framework was designed to meet several goals: high performance; efficient implementations for fundamental collections such as dynamic arrays, linked lists, trees, and hashtables; allowing different types of collections to work in a similar manner with high interoperability; and making it easy to extend or adapt a collection.
The entire collections framework is designed around a set of standard interfaces. Several standard implementations such as LinkedList, HashSet, and TreeSet are provided, and you may also implement your own collection.
LINKED LIST
Linked List is a part of the Collection framework present in java.util package. This class is an implementation of the LinkedList data structure, which is a linear data structure where the elements are not stored in contiguous locations. Every element is a separate object with a data part and an address part. The elements are linked using pointers and addresses. Each element is known as a node. Due to the dynamic nature and ease of insertions and deletions, linked lists are preferred over arrays. However, they have disadvantages: nodes cannot be accessed directly; instead, you need to start from the head and follow the link to reach a node.
There are three common types of linked lists: singly linked list, doubly linked list, and circular linked list.
Singly Linked List is a basic linked list type. It is a collection of nodes linked together sequentially where each node contains a data field and an address field which contains the reference to the next node.
Doubly Linked List adds a pointer to the previous node. Thus, you can go in either direction: forward or backward. Each node stores its data and has two links: one pointing to the previous node and one pointing to the next node. The first node has its previous link pointing to null, and the last node has its next node pointing to null.
Circular Linked List is a variation in which the last element is linked to the first element, forming a circular loop. Both singly linked lists and doubly linked lists can be made into circular linked lists.
HASH SET
The HashSet class implements the Set interface, backed by a hash table which is actually a HashMap instance. No guarantee is made as to the iteration order of the set, meaning the class does not guarantee a constant order of elements over time. This class permits the null element. The class also offers constant time performance for basic operations like add, remove, contains, and size, assuming the hash function disperses the elements properly among the buckets.
Important features of HashSet include: it implements the Set interface; the underlying data structure is Hashtable; duplicate values are not allowed; objects inserted are not guaranteed to be inserted in the same order; objects are inserted based on their hash code; null elements are allowed; HashSet is non-synchronized; it does not maintain insertion order; it is the best approach for search operations; the initial default capacity is sixteen, and the load factor is zero point seven five.
TREE SET
TreeSet is one of the most important implementations of the SortedSet interface in Java that uses a tree for storage. The ordering of the elements is maintained by a set using their natural ordering, whether or not an explicit comparator is provided. It is a non-linear data structure where data objects are organized in terms of hierarchical relationship. The structure is non-linear in the sense that, unlike a simple array, data in a tree is not organized linearly. Each data element is stored in a structure called a node. The topmost or starting node of the inverted tree is called the root node. All nodes are linked with an edge and form hierarchical subtrees beginning with the root node. A tree is a recursive data structure containing a set of one or more data nodes where one node is designated as the root, while the remaining nodes are called the children of the root. Nodes of a tree maintain a parent-child relationship, or they are sibling nodes. In a general tree, a node can have any number of children but only a single parent.
COLLECTION INTERFACE
The Collection interface is the root interface of the Java collections framework. There is no direct implementation of this interface. However, it is implemented through its subinterfaces like List, Set, and Queue.
Subinterfaces of Collection:
* List Interface – The List interface is an ordered collection that allows you to add and remove elements like an array.
* Set Interface – The Set interface allows you to store elements in different sets similar to a set in mathematics. It cannot have duplicate elements.
* Queue Interface – The Queue interface is used when you want to store and access elements in a First In, First Out (FIFO) manner.
COLLECTION CLASSES AND OPERATIONS
Java provides a set of standard collection classes that implement collection interfaces. Some classes provide full implementations that can be used as-is, and others are abstract classes providing skeletal implementations as starting points for creating concrete collections.
Basic operations include adding elements using the addAll method, sorting a collection using the sort method (ascending order) and reverseOrder method (descending order), searching in a collection using the binarySearch method (which returns the position of an object in a sorted list), copying elements using the copy method (which copies all elements from one list to another), and checking if two collections are disjoint (having no elements in common) using the disjoint method.
COLLECTION ALGORITHMS
The collections framework defines several algorithms that can be applied to collections and maps. These algorithms are defined as static methods within the Collections class. Several of the methods can throw a ClassCastException when an attempt is made to compare incompatible types, or an UnsupportedOperationException when an attempt is made to modify an unmodifiable collection.
Collections defines three static variables: EMPTY_SET, EMPTY_LIST, and EMPTY_MAP. All are immutable.

WEEKS 7-8: JAVA FX
INTRODUCTION
A graphical user interface (GUI) presents a user-friendly mechanism for interacting with an application. A GUI gives an application a distinctive look and feel. GUIs are built from GUI components, sometimes called controls or widgets.
INTRODUCTION TO JAVA FX
JavaFX is a set of packages that allow Java programmers to create rich graphics and media applications. Potential applications include GUI interfaces, 2D and 3D games, animations, visual effects, touch-enabled applications, and multimedia applications.
JAVA APIS
Java allows the developer to use three APIs for graphics and user interfaces: AWT (Abstract Windowing Toolkit), Swing, and JavaFX.
FEATURES OF JAVA FX
One major advantage of using JavaFX over other APIs is the integration of Scene Builder. JavaFX Scene Builder is a tool that allows a GUI to be developed without coding. It achieves this by providing a blank area where different components from JavaFX can be dragged and dropped to make different GUI forms. JavaFX Scene Builder generates FXML, which lets the designer make the interface separately from the program logic. FXML is an XML-based language. JavaFX Scene Builder can be used with any Java IDE to combine the program logic with the interface.
JAVA FX APPLICATION STRUCTURE
A JavaFX application is divided hierarchically into three main components known as Stage, Scene, and nodes. You need to import the javafx.application.Application class in every JavaFX application. This class provides lifecycle methods: init, start, and stop.
To create a basic JavaFX application, you need to import the Application class into your code, inherit the Application class into your class, and override the start method.
The Stage in a JavaFX application is similar to the Frame in a Swing application. It acts as a container for all the JavaFX objects. The primary Stage is created internally by the platform. Initially, the primary Stage looks empty, but you can add various objects to it. Objects can only be added in a hierarchical way: first, a scene graph will be added to the primary Stage, and then that scene graph may contain nodes. A node may be any object of the user interface, such as a text area, button, shape, or media.
The Scene actually holds all the physical contents (nodes) of a JavaFX application. The javafx.scene.Scene class provides all the methods to deal with a scene object. Creating a scene is necessary in order to visualize the contents on the Stage.
JAVA FX ARCHITECTURE
The JavaFX scene graph is the starting point for constructing a JavaFX application. It is a hierarchical tree of nodes that represents all of the visual elements of the application's user interface. It can handle input and can be rendered. A single element in a scene graph is called a node. Each node has an ID, a style class, and a bounding volume. With the exception of the root node of a scene graph, each node has a single parent and zero or more children. Nodes can also have effects such as blurs and shadows, opacity, transforms, event handlers (such as mouse, key, and input method), and application-specific state.
The javafx.scene API allows the creation and specification of several types of content, such as: nodes (shapes in 2D and 3D, images, media, embedded web browser, text, UI controls, charts, groups, and containers); state (transforms for positioning and orientation of nodes, visual effects, and other visual state); and effects (simple objects that change the appearance of scene graph nodes, such as blurs, shadows, and color adjustment).
The top layer of the JavaFX architecture provides a complete set of public APIs that support rich client application development. Below that, the graphics system is an implementation detail that supports both 2D and 3D scene graphs. It provides software rendering when the graphics hardware on a system is insufficient to support hardware-accelerated rendering. The Glass Windowing Toolkit is the lowest level in the JavaFX graphics stack; its main responsibility is to provide native operating services such as managing windows, timers, and surfaces.
The system runs several threads at any given time. The JavaFX application thread is the primary thread used by JavaFX application developers. Any live scene (a scene that is part of a window) must be accessed from this thread. The Prism render thread handles rendering separately from the event dispatcher, enabling concurrent processing. The media thread runs in the background and synchronizes the latest frames through the scene graph using the JavaFX application thread.
A pulse is an event that indicates to the JavaFX scene graph that it is time to synchronize the state of the elements on the scene graph with Prism. A pulse is throttled at a maximum of sixty frames per second and is fired whenever animations are running on the scene graph.
Additional features include: support for MP3, AIFF, and WAV audio files and FLV video files; a web component (a JavaFX UI control based on WebKit that provides a web viewer and full browsing functionality); CSS support for applying customized styling to the user interface without changing source code; UI controls that take advantage of the visually rich features of the JavaFX platform and are portable across platforms; layout containers (panes) for flexible and dynamic arrangements of UI controls; 2D and 3D transformations for each node in the scene graph; and visual effects such as drop shadow, reflection, and lighting to enhance the look of JavaFX applications in real time.
NETBEANS WITH JAVAFX SCENE BUILDER
NetBeans is an open-source, free IDE (Integrated Development Environment) which uses a modular architecture (it uses modules, which are groups of classes and libraries, to implement specific features into an application). NetBeans is one of the most popular Java IDEs. To start developing with NetBeans and JavaFX Scene Builder, you need to have both installed and JavaFX Scene Builder integrated with NetBeans.

LESSON 9: INTRODUCTION TO DATABASE
A database is a structured collection of data that is organized and stored for efficient retrieval and manipulation of information. Databases come in many different forms but have one thing in common: they have data that needs to be stored.
TYPES OF DATABASES
1. Relational Databases – The most common type of database. Uses a table structure to store data, with each table having a unique key that links to other tables.
2. Object-oriented Databases – Stores data in objects rather than tables. Commonly used in Java and Python.
3. NoSQL Databases – Stores data in a flexible, unstructured format.
RELATIONAL DATABASE
A relational database stores data in tables or relations. Each table has a unique identifier called a primary key. Columns represent attributes or fields of the data. Relationships between tables are defined through foreign keys, which link the tables together. Relational databases are used in various industries, including finance, healthcare, education, and many others.
Components of a Relational Database:
* Tables – The main structure that holds data. Tables consist of rows and columns: a row represents a record, and a column represents a field or attribute.
* Fields – Individual pieces of data that make up a record. Each field represents a single attribute or characteristic of a record, such as name, address, or phone number.
* Primary Keys – A unique identifier for each record in a table.
* Foreign Keys – A reference to a primary key in another table. Used to link data between tables and establish relationships.
* Indexes – Data structures that improve query performance by allowing data to be searched quickly.
* Constraints – Rules that govern the data in a table, ensuring that it is valid and consistent. Constraints are used for data integrity, preventing duplicates, and enforcing business rules.
Database Languages and Tools:
* Data Definition Language (DDL) – Used to create, modify, and delete database objects such as tables, indexes, and constraints.
* Data Manipulation Language (DML) – Used to insert, update, and delete data from tables.
* Query Language – Used to retrieve data from tables based on specific criteria.
* Transaction Management – Used to ensure data consistency and integrity, even in the event of system failures or other issues.
* Backup and Recovery – Tools used to create backups of the database and restore data in the event of data loss or corruption.
OBJECT-ORIENTED DATABASE
An object-oriented database stores data in the form of objects, similar to the objects used in object-oriented programming (OOP). It provides better support for complex data structures like hierarchies and networks than traditional relational databases. It is well-suited for applications such as computer-aided design systems, geographic information systems, and multimedia systems.
Example: In a library management system, you could represent each book as an object with attributes such as title, author, and publisher. Each borrower could also be represented as an object with attributes such as name, address, and phone number. Loans could then be represented as objects that connect books and borrowers, with attributes such as loan date, return date, and overdue fees. This object-oriented approach makes it easy to represent complex relationships and to perform queries such as "list all overdue books."
NOSQL DATABASE
A NoSQL database is a non-relational database that can handle large volumes of structured and unstructured data. It is horizontally scalable, meaning you can add more servers to increase capacity, as opposed to vertically scalable, where you need to upgrade the hardware on a single server.
Examples of NoSQL Databases:
* MongoDB – A popular document-oriented database that stores data in JSON-like documents. It is designed to be scalable and flexible, making it ideal for web applications. MongoDB has a rich query language that supports indexing, aggregation, and geospatial queries.
* Redis – A popular key-value store known for its speed and flexibility. Redis can store many types of data structures, including strings, hashes, and sets. It also supports advanced features like pub/sub messaging and Lua scripting.
* Apache Cassandra – A column-family store designed to handle large amounts of data across multiple servers. It is known for its high availability and fault tolerance, making it ideal for mission-critical applications. Cassandra has a query language called CQL that is similar to SQL.
* Neo4j – A popular graph database designed to manage relationships between data points. It uses a query language called Cypher that is optimized for graph traversal and pattern matching. Neo4j is commonly used for social networks, recommendation engines, and fraud detection.
IMPORTANCE OF DATABASES IN MODERN COMPUTING
Key aspects include data-driven decision-making, scalability and performance, security and privacy, and future trends in databases.

LESSON 11: JAVA NETWORKING
NETWORKING BASICS
* Internet Protocol (IP) – Every host has a unique IP address. It is more convenient to refer to a host using a hostname string. One hostname can correspond to multiple internet addresses.
* Domain Naming Service (DNS) – Maps names to numbers.
* java.net.InetAddress – This class converts between hostnames and internet addresses.
You can use InetAddress to get a host by name (such as a website or localhost), or to get the local host. You can also get an array of addresses if a hostname corresponds to multiple internet addresses, and then iterate through the array to print each host address.
PORTS
Many different services can be running on a host. A port identifies a service within a host. Standard port numbers are pre-assigned:
* Time of the day: 13
* FTP: 21
* Telnet: 23
* SMTP: 25
* HTTP: 80
An IP address combined with a port number is like a phone number for a service.
PROTOCOLS
Protocols are rules that facilitate communications between machines. They are standardized and documented so that machines can work with one another. Examples include HTTP (HyperText Transfer Protocol), FTP (File Transfer Protocol), SMTP (Simple Message Transfer Protocol), TCP (Transmission Control Protocol), and UDP (User Datagram Protocol).
CLIENT-SERVER INTERACTION
Communication between hosts is two-way, but usually the two hosts take different roles:
* Server – Waits for a client to make a request. The server is registered on a known port with the host (like a public phone number). It usually runs in an endless loop listening for incoming client connections.
* Client – Calls the server to start a conversation. The client uses the hostname or IP address and port number, sends a request, and waits for a response. Standard services (FTP, HTTP, SMTP, etc.) always have servers running on hosts using expected ports. The server offers shared resources (information, database, files, printer, computer power) to clients.
TELNET
Telnet assumes you want to connect to port 23 on the receiving host, because port 23 is where the telnet server is listening. An optional argument after the hostname allows you to connect to a different port. Many servers now refuse telnet connections due to security reasons.
SOCKET-LEVEL PROGRAMMING
Socket-level programming provides an abstraction of a bidirectional communication channel between hosts. You can send and receive data using streams.
WRITING CLIENTS
To write a client, you use the java.net.Socket class. You create a new socket with the hostname and port number of the connection. Then you call methods to get output and input streams for sending and receiving information.
For example, a socket client can make a connection to a time server and read the time that the server sends.
WRITING SERVERS
To write a server, you use the java.net.ServerSocket class. You create a new ServerSocket with a port number to listen on. Then you use the accept method to listen on the port. The accept method returns a socket when a client calls. You then call methods on that socket to get output and input streams.
An example is an echo server that sends a greeting message to the client and reads the client's response.
Many machines in computing environments may refuse socket connections due to security considerations. However, you can run servers on certain lab machines and connect from other machines on the same network.
MULTITHREADED SERVER
A multithreaded server starts a separate thread for each connection. This allows the server to handle multiple clients simultaneously. The server runs in an infinite loop, accepting incoming connections and spawning a new thread for each one. Each thread handles its own client communication independently.
COMMUNICATING WITH WEB SERVERS
Reasons for communicating with web servers include retrieving or sending information. You need to indicate the location of a resource using a Uniform Resource Locator (URL). A URL is a neat scheme for uniquely identifying all kinds of network resources. The basic form consists of a protocol, a sitename, and a pathname.
The class java.net.URL represents a Uniform Resource Locator. You can create a Java object that represents a URL. The URL class provides methods to get the host, path, port, and protocol.
The class java.net.URLConnection represents a communication link between the application and a URL. You can obtain a URLConnection from a URL. The typical steps for working with a URLConnection are: set properties of the connection, make the connection, query header information (such as content type, content length, content encoding, date, expiration, last modified), and then get input and output streams for reading and writing.
You can also directly open a stream for reading from a URL class. This method opens a connection to the URL and returns an input stream for reading from that connection.
SENDING INFORMATION TO WEB SERVERS
Web servers receive information from clients using either the GET or POST method.
* GET requests are made by browsers when the user types a URL on the address line, follows a link from a web page, or makes an HTML form that does not specify a method or specifically uses the GET method.
* POST requests are generated when an HTML form specifies the POST method.
When sending parameters to a CGI script using GET, you attach parameters to the end of the URL, separate parameters using ampersands, and encode parameters to avoid misinterpretation. This involves replacing spaces with plus signs and replacing each non-alphanumeric character with a percent sign followed by the hexadecimal code of the character. A disadvantage of GET is that long parameter strings might exceed browser limits.
When sending information using POST, you open a URL connection and send parameters using a stream. You set up the connection for output, get a stream for sending data, and then write the parameters. You are required to encode the parameters using a utility class that provides static methods for converting a string to the appropriate MIME format. The World Wide Web Consortium recommends using the UTF-8 encoding scheme.
